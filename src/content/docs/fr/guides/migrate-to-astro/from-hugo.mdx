[Hugo](https://gohugo.io) est un générateur de site statique open source construit avec Go. 

## Principales similitudes entre Hugo et Astro 

Hugo et Astro partagent certaines similitudes qui faciliteront la migration de votre projet : 

- Hugo et Astro sont tous deux des générateurs de site statique modernes, idéalement adaptés aux [sites web axés sur le contenu](/fr/concepts/why-astro/#content-driven) tels que les blogs. 

- Hugo et Astro vous permettent tous deux de [rédiger votre contenu en Markdown](/fr/guides/content/#markdown-authoring). Cependant, Hugo inclut plusieurs propriétés de frontmatter spéciales et vous permet de rédiger le frontmatter en YAML, TOML ou JSON. Même si bon nombre de vos propriétés de frontmatter existantes de Hugo ne seront pas "spéciales" dans Astro, vous pouvez continuer à utiliser vos fichiers Markdown existants et les valeurs du frontmatter YAML. 

- Hugo et Astro vous permettent tous deux d'améliorer votre site avec diverses [intégrations et packages externes](https://astro.build/integrations/). 

## Principales différences entre Hugo et Astro 

Lorsque vous reconstruisez votre site Hugo dans Astro, vous remarquerez quelques différences importantes : 

- Hugo utilise le templating Go pour la création de pages. La syntaxe d'Astro est un surensemble JSX de HTML, [comme indiqué ici](/fr/core-concepts/astro-components/). 

- Astro n'utilise pas de shortcodes pour le contenu dynamique dans les fichiers Markdown standard, mais [l'intégration MDX d'Astro](/fr/guides/integrations-guide/mdx/) vous permet d'utiliser JSX et d'importer des composants dans les fichiers MDX. 

- Alors que Hugo peut utiliser des "partials" pour des éléments de mise en page réutilisables, [Astro est entièrement basé sur des composants](/fr/core-concepts/astro-components/). Tout fichier `.astro` peut être un composant, une mise en page ou une page entière, et peut importer et rendre n'importe quel autre composant Astro. Les composants Astro peuvent également inclure [d'autres composants de framework UI (par exemple, React, Svelte, Vue, Solid)](/fr/core-concepts/framework-components/) ainsi que du contenu ou des métadonnées provenant [d'autres fichiers de votre projet](/fr/guides/imports/), tels que Markdown ou MDX. 

## Passer de Hugo à Astro 

Pour convertir un blog Hugo en Astro, commencez avec notre modèle de démarrage de thème de blog, ou explorez davantage de thèmes de blog de la communauté dans notre [vitrine de thèmes](https://astro.build/themes/). 

Vous pouvez transmettre un argument `--template` à la commande `create astro` pour démarrer un nouveau projet Astro avec l'un de nos démarreurs officiels. Ou vous pouvez [démarrer un nouveau projet à partir de n'importe quel référentiel Astro existant sur GitHub](/fr/install/auto/#starter-templates).

Amenez vos fichiers Markdown existants (ou MDX, avec notre intégration facultative) en tant que contenu pour [créer des pages en Markdown ou MDX](/fr/guides/markdown-content/). Vous devrez peut-être convertir votre frontmatter en YAML, car Astro n'accepte que le frontmatter YAML dans ces fichiers. 

Pour continuer à utiliser du contenu dynamique tel que des variables, des expressions ou des composants UI dans votre contenu Markdown, ajoutez l'intégration facultative MDX d'Astro et convertissez vos fichiers Markdown existants en [pages MDX](/fr/guides/markdown-content/). MDX prend en charge le frontmatter YAML, vous permettant de conserver vos propriétés de frontmatter existantes. Cependant, vous devez remplacer toute syntaxe de shortcode par [la syntaxe propre à MDX](https://mdxjs.com/docs/what-is-mdx/#mdx-syntax), qui autorise les expressions JSX et/ou les importations de composants. 

Pour convertir d'autres types de sites, tels qu'un portfolio ou un site de documentation, explorez davantage de modèles de démarrage officiels sur [astro.new](https://astro.new). Vous trouverez un lien vers le référentiel GitHub de chaque projet, ainsi que des liens en un clic pour ouvrir un projet fonctionnel dans les environnements de développement en ligne StackBlitz, CodeSandbox et Gitpod.
